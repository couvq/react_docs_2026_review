* The web supports creating web pages with HTML, CSS, and Javascript. HTML can be thought of like a skeleton, providing markup for a page with tags like `<h1>`, `<section>`, etc. in order to form a website's content and structure. CSS can be thought of like skin, hair, or other defining features and serves the purpose of adding style to a webpage. Javascript can be thought of like muscles and the nervous system and serves the purpose of adding interactivity to a web page. React takes these base technologies a step further, and allows you to combine these into reusable components using a syntax called JSX. Using component based libraries and frameworks can speed up your development and make the UI easier to understand with the resuability of components.
* Components must start with a captial letter. This is so React can determine what is a component, and what is just regualar markup. For example, `<Section>` is seen by React as a component, while `<section>` is seen by React as markup.
* Components are just a special type of Javascript function that return JSX.
